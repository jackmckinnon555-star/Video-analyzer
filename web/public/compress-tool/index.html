TRA Video Analyzer · desktop uploader

Upload big videos with one click

The in-browser compressor can't reliably handle multi-GB files. This small script does the work on your computer — compresses, uploads, and opens the results page — in about the time it takes to get coffee.

  1. Install ffmpeg if you haven't. 
    # Windows
winget install Gyan.FFmpeg

# macOS
brew install ffmpeg

# Linux (Debian / Ubuntu)
sudo apt install ffmpeg

    Verify: open a terminal and run ffmpeg -version.

  2. Download the uploader script:
    upload.ps1 Windows · PowerShell upload.sh macOS / Linux / Git Bash README.txt Full instructions
  3. Run it against your video.

    Windows (PowerShell)

    # Double-click the script, paste the path it asks for
# OR pass the file path directly:
.\upload.ps1 "C:\path\to\video.mp4"

# If PowerShell blocks unsigned scripts:
powershell -ExecutionPolicy Bypass -File .\upload.ps1 "video.mp4"

    macOS / Linux / Git Bash

    chmod +x upload.sh
./upload.sh path/to/video.mp4

    First run prompts for the site password — it's cached in %APPDATA%\video-analyzer\config.json (or ~/.config/video-analyzer/config.json) so you only enter it once.

  4. That's it.

    The script compresses → uploads → finalizes → opens the results page in your browser. Processing takes another 2-5 minutes on the server side; your browser tab shows live status.

Native ffmpeg runs at 2-5× realtime — a 3-hour video compresses in 30-90 seconds. Upload is another 10-30 seconds. You're usually done in under 2 minutes.

Prefer a one-liner?

These match what the script produces (audio-only; fits any length): 

ffmpeg -i input.mp4 -vn -c:a aac -b:a 32k -ac 1 -movflags +faststart output.m4a

For short clips where you want playable video:

ffmpeg -i input.mp4 \
  -c:v libx264 -preset ultrafast -b:v 300k -maxrate 300k -bufsize 600k \
  -vf "scale='min(640,iw)':'-2'" \
  -c:a aac -b:a 32k -ac 1 \
  -movflags +faststart output.mp4
Troubleshooting

"ffmpeg not found" — the installer didn't put it on PATH. Restart your terminal. On Windows, sometimes you need to sign out and back in.

"Invalid site password" — the script cleared the bad password from your config file. Run it again and re-enter.

Windows SmartScreen blocked the .ps1 — right-click the file → Properties → check "Unblock" at the bottom → OK → retry.

Output is still >50 MB — the source is extraordinarily long (>10 hours) or dense. Trim it first:
ffmpeg -i input.mp4 -t 3600 -c copy part1.mp4

I'd rather compress in the browser — the experimental browser compressor still exists. It may not work for multi-GB files.
