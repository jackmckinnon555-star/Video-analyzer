How it works

  1. Install and launch.

    Creates a shortcut on your desktop and in the start menu / Applications folder.

  2. Enter the site password once.

    Cached locally; you won't be asked again on this computer.

  3. Drop a video file on the window.

    Compresses with native ffmpeg (2-5× realtime), uploads to the server, opens the results page in your browser. A 3-hour video is usually done in under 2 minutes.

Prefer the no-install script?

If you already have ffmpeg installed and would rather drive this from a terminal, grab the script version. Same output, same one-command flow.

Install ffmpeg first

# Windows
winget install Gyan.FFmpeg

# macOS
brew install ffmpeg

# Linux (Debian / Ubuntu)
sudo apt install ffmpeg

Download the script

upload.ps1 Windows · PowerShell upload.sh macOS / Linux / Git Bash README.txt Full instructions

Run it

# Windows
.\upload.ps1 "C:\path\to\video.mp4"

# macOS / Linux
chmod +x upload.sh
./upload.sh path/to/video.mp4

First run prompts for the site password — cached in %APPDATA%\video-analyzer\config.json or ~/.config/video-analyzer/config.json.

Prefer a raw ffmpeg one-liner?

Matches what the installer and scripts produce. Upload the resulting file through the homepage uploader. 

ffmpeg -i input.mp4 -vn -c:a aac -b:a 32k -ac 1 -movflags +faststart output.m4a

For short clips where you want playable video:

ffmpeg -i input.mp4 \
  -c:v libx264 -preset ultrafast -b:v 300k -maxrate 300k -bufsize 600k \
  -vf "scale='min(640,iw)':'-2'" \
  -c:a aac -b:a 32k -ac 1 \
  -movflags +faststart output.mp4
Troubleshooting

Installer download is blocked — your browser or antivirus may flag unsigned installers. Use the GitHub Releases page directly: releases/latest.

App opens but "site unreachable" — check your internet. The app talks to video-analyzer-tra.netlify.app over HTTPS.

Invalid site password — the app clears the bad password automatically. Re-enter when prompted.

"ffmpeg not found" (scripts path only) — restart your terminal after installing. On Windows, sometimes you need to sign out and back in to refresh PATH.

Output is still too big — the source is extraordinarily long (>10 hours) or dense. Trim it first:
ffmpeg -i input.mp4 -t 3600 -c copy part1.mp4
